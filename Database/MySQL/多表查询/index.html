【MySQL】多表查询

发表于 2026-06-07 分类于 Database ， MySQL 阅读次数：
本文字数： 2.1k 阅读时长 ≈ 4 分钟
多表查询理论，以及如何高效的在不同表之间查询数据

多表关系

各个表之间存在着各种联系，一般分为三种，与他们的实现：

  • 一对多/多对一：在多的一方建立外键，指向一的一方的主键
  • 多对多：建立第三张中间表，中间表至少包含两个外键，分别关联两方主键
  • 一对一：多用于单表拆分，在任意一方加入外键，关联另一方的主键，并设置外键为UNIQUE

多表查询

  • SELECT * FROM a, b;可以查询到a表和b表的笛卡尔积，要想获得有效的查询，我们需要消除多余的笛卡尔积

  • SELECT * FROM a, b WHERE a.id = b.id;进行了等值连接，仅查询了两个表的关联数据

分类

  • 连接查询
    • 内连接：查询a，b表的交集部分数据
    • 左外连接：查询左表所有数据，以及两表交集部分数据
    • 右外连接：查询右表所有数据，以及两表交集部分数据
    • 自连接：当前表与自身的连接查询，必须使用别名

内连接

  • 隐式内连接：SELECT 字段列表 FROM 表1, 表2 WHERE 条件...;

  • 显式内连接：SELECT 字段列表 FROM 表1 [INNER] JOIN 表2 ON 连接条件...;

e.g. 查询员工表内员工姓名及关联的部门名称（分为员工表emp和部门表dept）

外连接

  • 左外连接：SELECT 字段列表 FROM 表1 LEFT [OUTER] JOIN 表2 ON 连接条件...;
  • 右外连接：SELECT 字段列表 FROM 表1 RIGHT [OUTER] JOIN 表2 ON 连接条件...;

e.g.1 查询emp表的所有数据和对应的部门信息(dept表)

e.g.2 查询dept表的所有数据和对应的员工信息(emp表)

自连接

SELECT 字段列表 FROM 表A 别名A JOIN 表B 别名B ON 条件...

自连接可以是内连接查询，也可以是外连接查询

e.g. 查询员工表内员工及其所属领导的名字（所属领导id和员工id在同一张表emp上）

联合查询

把多次查询的结果合并起来，形成一个新的查询结果集

  • UNION合并并去重
  • UNION ALL只会合并

条件：联合查询中多张表的列数、字段类型必须一致

子查询

嵌套SELECT语句，成为嵌套查询或子查询

外部语句可以是SELECT/INSERT/UPDATE/DELETE中的任意一个

分类

根据结果：

  • 标量子查询：子查询结果为单个值
  • 列子查询：子查询结果为一列
  • 行子查询：子查询结果为一行
  • 表子查询：子查询结果为多行多列

根据位置：

  • WHERE之后
  • FROM之后
  • SELECT之后
